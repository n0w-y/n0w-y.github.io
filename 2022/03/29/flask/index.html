flask-migrate

flask中一般使用SQLAlchemy框架操作数据库，db.create_all()创建数据库，在创建数据库后若有对数据库结构进行修改的需求，则需要将原数据库进行删除再创建，显然这不是我们期望的。flask-migrate就提供数据迁移的功能，能够随时更新数据库字段

数据库操作

初始化 python manage.py db init

只有第一次需要进行初始化，初始化后会增加一个migrations文件夹，该文件夹下的versions文件夹存放着数据库的版本，此时创建了迁移仓库。

迁移 python manage.py db migrate

生成迁移文件，此时并未创建表

更新 python manage.py db upgrade

更新数据库，创建相应的表

帮助 python manage.py db --help

查看帮助

Bootstrap Grid System

Bootstrap是基于html、css、js的开源前端框架，使用Bootstrap能够快速地开发美观页面

而Grid System是Bootstrap的响应式、移动设备优先（即优先设计小设备）的网格系统，Grid System可以根据设备或视口大小变换布局

在Grid System中页面最多能够被分为12列，通过定义行占多少列来实现布局，布局必须放在container内，创建row然后用col-xx-x定义行宽为多少列

何为响应式？

响应式的目的是让用户在各种尺寸的设备上浏览时都能有良好的视觉效果，上述举例并没有很好地表现响应式效果，再展示一下

可以通过在拉伸和压缩浏览器宽度来观察效果

当容器宽度大于等于1200px时，每条内容占12列

当容器宽度大于等于992px时，每条内容占6列，一行共两条内容

当容器宽度大于等于768px时，每条内容占3列，一行共四条内容

当容器宽度小于等于768px时，每条内容占1列，一行最多十二条内容

另外，还可以通过offset-xx-x设置列偏移
