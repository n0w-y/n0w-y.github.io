DHCP攻击与防御

什么是DHCP

DHCP-动态主机配置协议（即插即用协议、零配置协议）

当一个主机新加入一个网络时，DHCP允许主机自动获取IP地址，同时还能获取子网掩码、默认网关和本地DNS服务器地址，即DHCP具有将主机自动连接进一个网络的能力。若没有DHCP，则每有一台主机加入网络，就需要管理员手动为其配置IP

如何工作

主机获取ip可分为四个步骤：DHCP Discover、DHCP Offer、DHCP Request、DHCP Ack

DHCP Discover

主机新加入某个网络后，生成DHCP Discover报文，DHCP Discover报文封装在UDP数据报中，然后封装在IP数据报中，使用目的地址为广播地址255.255.255.255，源IP地址为0.0.0.0，最后在数据链路层封装成帧，以广播形式发送

DHCP Offer

DHCP服务器收到DHCP Discover报文时，生成DHCP Offer报文，DHCP Offer报文包含推荐IP、网络掩码和IP租用期，发送报文源IP地址为DHCP服务器的IP，目的IP地址为255.255.255.255

DHCP Request

有可能存在多个DHCP服务器，所以主机可能会收到多个DHCP Offer报文，此时主机选择第一个收到的提供报文，丢弃其它的DHCP Offer报文。主机发送DHCP Request报文，仍以广播形式发送但报文中存在指定的DHCP服务器ID，告知该服务器将要租用提供的IP

DHCP Ack

DHCP服务器收到DHCP Request报文后发送DHCP Ack报文，告知主机可以使用提供的IP地址，并在IP地址池标记分配的IP地址，在服务端将MAC地址和IP地址绑定

IP地址续租

当租期50%过后，客户机会再次发送DHCP Request包，进行续约，如服务器无响应，则继续使用并在 87.5%再次 DHCP Request包，进行续约，如果没有，则客户端继续使用该IP地址直至到期。客户端向DHCP 服务器发送DHCP Release报文来释放该IP地址，并开始新的IP地址申请过程。

DHCP攻击

仿冒DHCP报文攻击

攻击者向DHCP Server申请大量的IP地址或发送DHCP Request 数据包申请续租IP地址，导致IP池中IP地址被耗尽，新上线主机无法获得IP地址，或攻击者冒充合法用户向DHCP Server发送DHCP Release报文，导致用户异常下线

DHCP Server仿冒攻击

由于DHCP Server和DHCP Client之间没有认证机制，所以攻击者可以伪装成DHCP Server，就可以为客户端分配IP地址以及其他网络参数。攻击者能够通过为DHCP客户端分配错误的信息导致其无法上线（可通过饿死攻击，在与DHCP服务器的竞争中胜利）

中间人攻击

攻击者将自己的MAC地址和DHCP服务器IP地址构造一个DHCP报文发送给DHCP客户端，客户端收到后学习，导致DHCP客户端发送到DHCP的所有数据包都经过攻击者

攻击者将自己的MAC地址和DHCP客户端的IP地址构造成一个DHCP报文发送给DHCP服务器，DHCP服务器收到后学习MAC和IP地址对应关系，导致DHCP服务器发送给DHCP客户端的所有数据包都经过攻击者

DHCP防御

要防御DHCP攻击，只要让非授权的DHCP服务器的回应无法通过网络即可。交换机启用DHCP Snooping技术，通过信任端口和非信任端口、DHCP Snooping绑定表便能只让合法的DHCP回应通过交换机，此时网络中的PC不用做出任何改变

DHCP Snooping

DHCP Snooping 是一种DHCP安全功能，类似于在DHCP Server 和DHCP Client之间的防火墙。

在DHCP Snooping技术中，交换机的端口分为信任端口和非信任端口，默认所有端口都为非信任端口，通常会将与DHCP Server相连的端口配置为信任端口，其它端口为非信任端口。交换机从信任端口收到DHCP报文后会正常转发，从非信任端口收到DHCP Server报文后会直接丢弃

具有DHCP Snooping 功能的设备通过DHCP Ack报文创建、维护DHCP Snooping绑定表，绑定表中记录了主机的IP地址、IP地址租用时间、MAC地址、VLAN ID 和接口信息，设备通过DHCP Snooping绑定表检查DHCP报文的有效性

仿冒DHCP报文攻击防御

对于伪造DHCP Request 和 DHCP Release报文

通过DHCP Snooping绑定表对DHCP Request和DHCP Release报文进行检查（VLAN、IP、MAC、接口信息是否匹配动态绑定表），若信息不匹配则丢弃

对于伪造DHCP Discover报文

为了抑制大量恶意申请IP地址，可以配置设备或接口允许接入的最大DHCP用户数，当接入用户达到阈值时不再允许任何用户通过此设备或接口申请IP地址

DHCP Server仿冒攻击防御

通过DHCP Snooping技术中的交换机的信任端口、非信任端口，对于非信任端口的DHCP Server报文会直接丢弃，从而防止DHCP Server仿冒攻击

中间人攻击防御

DHCP中间人攻击需要利用ARP欺骗，若能够防御ARP欺骗那么也能给防御DHCP中间人攻击

运行DHCP Snooping的交换机会创建并维护DHCP Snooping绑定表，当攻击者发送ARP请求报文时交换机会检查报文中的源IP地址和源MAC地址，若不匹配则丢弃该报文
